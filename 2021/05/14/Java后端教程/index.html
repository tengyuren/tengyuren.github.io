一、JavaWeb

1、基本概念

  • 静态资源：html，css，数据不会发生变化
  • 动态资源：数据会发生变化，在不同时间、地点看好的信息不同（技术栈：Serviet/JSP，ASP，PHP）

动态资源开发的技术统称为javaweb。

2、web服务器

2.1 tomcat 安装

tomcat服务器：运行JSP和Servlet， 获得请求返回响应。

  • bin:启动，关闭的脚本文件

  • conf：配置文件

  • libs：依赖的jar包

  • webapps：存放网站

2.2 启动Tomcat

bin目录文件下有一个startup.bat文件，启动。默认端口8080

shutdown.bat关闭Tomcat

可能遇到的问题：java环境变量、闪退问题（需要配置兼容性）、乱码问题(配置文件中修改)

2.3 Tomcat配置

server.xml => 核心配置文件

配置主机的端口号、配置主机的名称（默认的主机名称为locahost，默认网站应用的存放位置为webapps）

面试题

请你谈一谈网站时如何进行访问的？

  • 输入一个域名，客户端进入hosts配置文件，检查有没有这个域名的映射（例如localhost），如果有则直接返回对应的IP地址，如果没有则去DNS服务器访问获得对应域名的IP地址

2.4 发布一个简单的网站

直接放到webapps文件夹中，通过http://localhost:8080/name访问就行。

二、Spring

1、基本知识

Spring是一个轻量级控制反转（IOC）和面向切面（AOP）的容量框架

2002年推出了Spring框架的雏形interface21 ， 2004 年推出Spring 1.0

Rod Johnson：Spring Framework的创始人，音乐教授，不要重复造轮子

理念：保持强大的向后兼容性，针对简化企业级开发，本身是一个大杂烩，整合了现有的技术框架。

SSH： Struct2+Spring+Hibernate

SSM: SpringMVC+Spring+Mybatis

优点：开源的、免费的框架、轻量级的、非入侵式的（引入Spring不会改变原来项目代码）、支持事务的处理、对框架整合的支持，控制反转（IOC）、面向切面编程（AOP）

2、组成

SpringBoot：一个快速开发的脚手架、开发微服务、约定大于配置

SpringCloud：基于SpringBoot实现的，集合SpringBoot服务

弊端：发展了太久之后违背了以前的理念，到后期配置十分繁琐

3、IOC

理论： 程序不再具有主动性、而是被动的接收

控制；谁来控制对象的创建，传统应用程序的对象是由程序本身控制创建的，使用Spring后，对象是由Spring来创建的。

反转：程序本身不创建对象，而是被动的接受对象

依赖注入：利用set的方法进行注入

不需要去程序中改动，要实现不同的操作只需要在xml配置文件中进行修改，一句话：对象由Spring来创建、管理、装配。

6、AOP

6、1 设计模式：

代理模式：Spring AOP的底层

分类：

  • 静态代理

抽象角色：一般会用接口或者抽象类实现（租房的服务、增删改查）

真实角色：被代理的角色（房东、真正的增删改查）

代理角色：代理真实角色，代理后会做一些附属操作（中介、在增删改查之前加入日志）

客户：访问代理对象的人（租房人）

好处：可以使真实角色的操作更加纯粹，不用关注公共业务，公共业务交给代理角色，方便公共业务的维护和集中管理。

缺点：一个真实角色会产生一个代理角色，代码量会增加。

  • 动态代理

动态代理和静态代理角色一样，动态代理的代理类是动态生成的，不是直接写好的。

  • 基于接口–JDK的动态代理
  • 基于类的–cglib
  • java字节码实现：JAVAssist

需要了解两个类：Proxy 、 InvocationHandler
